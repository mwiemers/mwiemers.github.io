Sentiment analysis¶

Sentiment analysis refers to techniques used to assess if a text contains positive or negative affect or attitudes. We will use a sentiment analyser from NLTK called VADER (Valence Aware Dictionary and sEntiment Reasoner). It is an open source, rule based tool "specifically attuned to sentiments expresses in social media".

What rule means here is two-fold. Firstly, tokens are annotated with polarity scores. For example, ":)" might add 0.2 to the positive score, and the word "hate" might add 0.4 to the negative score.

Secondly, these additions are conditional on structural information, such as negation or degree modifiers. For example, "not great" would count as something negative and "excellent!" would be more positive than just "excellent".

Of course, these rules will still miss some double meaning, irony or idioms, as language is too complex to classify these through rules. In the appendix, we discuss modern approaches to solving these problems. Nevertheless, lightweight analysers like VADER are sufficient for many applications.

More concretely, suppose we are given a piece of text. This could be any ordered collection of words e.g. multiple sentences associated with some named entity or an extracted topic (we will see details of these later). Here be aware of cleaning the text: we would like to ideally leave the raw version so that we keep information such as uppercase words or punctuation. Running VADER gives us four scores.

  • The compound score is an average of words' sentiment scores adjusted according to rules that account for semantic information such as word order, degree modifiers (e.g. very, a lot), punctuation amplifiers (e.g. !), negation, contrasts. The output is normalized to fall between -1 and 1. This is the most useful and most popular metric for measuring overall sentiment of a piece of text.
  • The pos, neu and neg scores are proportions of text that fall in the respective category, so they add up to 1. These don't take into account any additional rules besides the raw word classification. This is supplementary information, useful for analysing style of how the sentiment is conveyed. This information should be avoided for measuring overall sentiment, where the compound score is preferred.

Applications of sentiment analysis with VADER include hate speech detection and content recommendations. See Semantic Scholar for more papers citing VADER. There are many other sentiment analysers. Some simple, like AFINN, a list of valence scores per word.

Around 2016, benchmarks were incorporating many rule-based analysers (see SentiBench). Due to recent advancements in deep neural network architectures for natural language processing, nowadays benchmarks are usually occupied by fine-tuned models of this kind. For example, see a recent analysis of contact-tracing reviews.

That said, VADER is often good enough: it's much simpler conceptually, and much cheaper to run. It is also validated by real people in the context of sentiment analysis of social media, so might outperform modern methods in that sense.

Topic modelling¶

Topic modelling is an umbrella term for models that extract a set of "topics" from a collection of "documents". A topic is a collection of tokens; a pattern of word use, a groups of terms that tend to occur together in documents. A document is anything that is a single piece that is also a part of a larger corpora of similar pieces. No semantic information is used at this point: these are statistical models dealing with co-occurence of phrases.

For example, a document can be an article among articles authored by someone, or a chapter among other chapters in a book, or a sentence among sentences attributed to someone. Topic modelling can be applied on anything of this form, but it helps if it makes sense as a collection of documents produced by some entity. Note that multiple documents are necessary: one article won't do, we need either to have a collection of articles, or to instead look at e.g. sentences in that article. Of course, the former is preferred to the latter, as sentences might not contain enough words for the topics to be robustly extracted. Be aware that this will not fail explicitly; the topics will be computed without errors, they just might not be insightful.

One of the most widely used models for topic modelling is the so-called Latent Dirichlet Allocation (LDA). LDA is available to use through many software packages, including gensim, our choice for this tutorial. Apart from needing the input as described above (set of text documents), LDA also makes a statistical assumption that there is a fixed number of topics across documents and that each document exhibits the topics to varying degree. These are of course fairly subjective; we should simply be critical of LDA's output for documents that might not comply with these assumptions. In the earlier example of using sentences as documents, one might argue that many sentences might not exhibit any relevant topics at all.

Suppose we have a corpora of articles about pets, and we run LDA for 4 topics. Slightly changing number of topics should not change prominent topics, but it is hard to predict this in general. We will see later how one could decide on the number of topics to use in a final model. For now, we stick to 4. We inspect computed topics and see

Here is a more graphical characterisation of topic models: documents live on a "probability simplex", which in the case of three topics is a triangle. A hypothetical document located stricly on the red dot would have 100% of the topic 2 and 0% for both topic 1 and 3. This leads to the notion of the "latent space": we can find topic distribution for a new document. If we consider documents indexed by time, we can think of this as prediction of the topics in a future document.

Evaluation and tuning of topic models¶

We will need two components for this experiment

  1. Some (possibly relative) measure of how good the model is given fixed parameters
  2. Splititng of the data into training and test: we want to compute the aforementioned measure on data that the model haven't "seen"

Let us first split our corpus into 25% test and 75% train set. We use the sample function to create a list with the indices for the test set. The first argument is the indices to sample from. The second argument is the number of elements to sample from. The double forward slash performs integer division.

In [ ]:

LdaModel has some parameters we can adjust to get better results. Two main ones are the number of topics to find and the number of iterations to compute the model for. The default values are 100 and 50, respectively. We consider five models below.

# 5 topics, 1000 iterations
m5 = LdaModel(bows_train, id2word=dct, num_topics=5, iterations=1000)

From the technical perspective, Bayesian framework LDA is written in gives us a straightforward measure called the model evidence, which is the likelihood of the data given the learned model parameters. It is called perplexity in the topic modelling context and roughly measures how likely are the documents in test given model trained on documents in train. It was observed that it ralely correlates with human judgement on the quality of learned topics. Second option is the so-called topic coherence, which aims to use more contextual information than perplexity. See more details here.

Both perplexity and coherence, like $R^2$ in linear models, are useful for relative comparison of models, but discouraged as an objective evaluation of a model. Ideally, we should only compare models with different parameters that are developed on the same data. In practice, topic coherence does provide some qualitative information about how topics are constructed; one can think of it as a measure of the relative distance between words within a topic. The closer they are, the more "coherent" the topics should be. Averaging out over topics, we get a single number for a model, and the smaller it is, the better.

LdaModel instance has a log_perplexity method that takes the hold-out test set from bag-of-words corpus. The higher the better.

What are topics useful for?¶

Topics as explanations¶

Inaugural speech topic modelling example was not very successful, even after tuning. It's a real world data; we didn't have that many examples. One could make an argument that an inaugural speech has too much noise and is written to drive human emotions, and not necessarily communicate information. For these kind of texts, human specialists are still necessary, as in this analysis from Google News Lab of the same problem.

But of course, sometimes topics are meaningful. Data from social media tends to be especially effective: character limits and large number of examples make the problem much more tractable. One recent study of Caldern et al. (2020) used LDA to look into hate speech against immigrants on Twitter. In this case, separate topics such as "lack of control of illegal immigration" or "economic assistance to immigrants" were found.

Topics as features¶

We saw that we can exprapolate the fitted topic model to predict topics in a new document. Another way to use topics, irrespective whether they were found for a whole corpus or extrapolated to new documents from some existing corpus, is to use them as features to other statistical models. For example, suppose we would like to classify tweets as angry or calm. We could represent the tweets with columns such as number of words, sentiment score and the weight of each topic. Logistic regression might be not enough for this problem, but we could choose any other model. Another interesting example is clustering Shakespear using LDA topics in the second part of this data driven literary analysis.

Chang et al. (2009) conducted a large-scale human study on interpretability of topics i.e. how meaningful the latent space is. Suprisingly, good scores of predictive potential (such as perplexity and coherence) were found to be inversely correlated with the semantic potential (how useful are the topics for explanatory analysis). This point should not be taken as a principle against neither use of topic modelling, but rather an encouragement to experiment and not pay too much attention to the scores! Tuning the model might be a good place to start, but shouldn't be a big factor in final modelling decisions.

Appendix¶

More technical details of LDA¶

LDA is an example of a hierarchical model where static collections of discrete units of data (documents made up of words) are represented as random mixtures over some latent variables (topics) where each such latent variable itself is represented as a distribution over over the units (words). The main innovations of LDA is parametrization that allows for having a document represented by multiple topics a not just one topic (as would be the case in ordinary clustering) and the use of Dirichlet distribution to capture the intuition that a topic should only consist of a relatively small number of words. See the seminal paper Blei et al. (2003) for details on applications of LDA. For even more details, see this website. Note that LDA can be used for any collection of discrete data! In fact, it was originally used in genetics for population modelling. Here is some useful further reading to bridge the gap between formal describtion and applied use

  • Probabilistic Topic Models (2012) is a nice semi-technical survey of topic modelling, with a focus on LDA
  • Topic Modelling Made Just Simple Enough (2012) is a nice introduction non-mathematical to the Bayesian perspective of LDA in the context of digital humanities; the author interprets topic as discourse, and poses that straightforward topics are not necessarily most interesting!
  • Compare and Contrast (2011) gives some more perspective on topic modelling for digital humanities

There are also other topic coherence measures, such as c_v (for this one, higher the better). The one that we used, the intristic topic coherence u_mass, is computed as follows.

  1. Fix a topic $T=\{w_1,\ldots,w_N\}$ made of $N$ words
  2. $D(w_i)$ the number of documents with word $w_i$ in them
  3. $D(w_i, w_j)$ the number of documents with both words $w_i$ and $w_j$
  4. Define a score $S(w_i, w_j)=\log \frac{D(w_i, w_j)+1}{D(w_i)}$
  5. Coherence of $T$ is then simply $\displaystyle\sum_{i<j} S(w_i, w_j)$

Deep Learning and NLP¶

What we saw so far leaves an impression that quantitative text analysis amounts to either hand-annotated databases driven by linguistic theory (such as WordNet) or extracting information from phrase occurence patterns with statistics. The former is not only expensive to maintain, but also does not offer much insight beyond well-behaved contexts. The latter makes no use of the rich context and semantics, and so will miss out on the likes of irony, double meaning or idioms. But as mentioned in the section on sentiment analysis, best results are obtained using an entirely different methodology.

Neural network is a high-dimensional (many more parameters than e.g. linear regression) model that can be made arbitrarily complex (deep). It represents some function of the data and is trained on annotated examples so that this function maps to the right labels. They are much more expensive to compute than simpler models like linear regression, but are far superior in generalizing to unseen data points. A significant realization was that we can use these models to force our data into some lower-dimensional representation and work with that, often with more deep neural networks. With text, it works roughly as follows

  • Enumerate unique tokens and index them by 1, 2, 3, ... (as with processing for topic modelling)
  • Assign to each token a vector with 1 on the index of that token in the dictionary and 0 otherwise
  • Compress these to real-valued vectors in a lower-dimensional space, e.g. 200 dimensions
    • Recall that in a vector space gives a notion of closeness (with respect to a norm)
    • Create buckets of words close to each other, and set the network so that they are "close" in the target vector space
    • We might see that things like king + woman = queen happen; we extracted some contextual information!
    • Word2vec was the first model of this, now there are many more
  • Build another deep model, this time using the dense vector representation as input

Another big revolution in NLP were a kind of deep neural models called transformers, introduced by the 2017 paper Attention is All You Need. Next major discovery was that language models are few-shot learners, which means that we can compute these huge transformer-based models on some large text datasets, and then reuse them in new contexts by giving them a small number of examples from the new context (also known as fine-tuning).

These models are decent even without fine-tuning. We call these tasks zero-shot, as there are no new context-specific examples we show to the model. We will focus on zero-shot classification. We provide some labels, like vacation, work, school, and text will be assigned confidence level for each label. There is a nice package with many recent language models, called transformers. You need to install it manually, along with torch.

More concretely, we load the Facebook's BART model trained on Multi-Genre Natural Language Inference (MultiNLI) dataset. NLI models assign to a pair of sentences a label: entails, contradict or neutral. It is then adapted for zero-shot classification task by adding a sentence "This text is about LABEL" to make a pair with an input sentence. The relation between the two sentences is then posed by BART, and the confidence translates to classification confidence for a given label.

bart_mnli = pipeline("zero-shot-classification", model="facebook/bart-large-mnli")
